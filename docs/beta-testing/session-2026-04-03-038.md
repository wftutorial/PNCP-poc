## ISSUE-071 — ABERTO (P3)

**Persona:** Jordan "The Paranoid"
**Severidade:** P3
**Área:** Buscar — QuotaCounter inline (abaixo dos resultados)
**Data:** 2026-04-03
**Status:** ABERTO

### Passos para Reproduzir
1. Fazer login como usuário com plano ativo (não admin)
2. Realizar qualquer busca com resultados
3. Observar o contador de quota abaixo dos resultados: "X análises restantes neste mês"
4. Observar o label à direita: **"Renovação: 01/05/2026"**

### Comportamento Observado
Label `"Renovação: ${formattedResetDate}"` aparece no QuotaCounter com a `quota_reset_date` (1º do próximo mês para usuários regulares). Para admin mostra "Renovação: 03/04/2026" (hoje). O mesmo problema semântico de ISSUE-070, mas em componente diferente.

### Comportamento Esperado
Label deveria dizer `"Reset em: ${formattedResetDate}"` para ser semanticamente preciso: é reset de cota mensal, não renovação de assinatura Stripe.

### Notas Técnicas
- **Arquivo:** `frontend/app/components/QuotaCounter.tsx:96`
- **Linha exata:** `` `Renovação: ${formattedResetDate}` `` → deve ser `` `Reset em: ${formattedResetDate}` ``
- ISSUE-070 fix alterou `/conta/plano` mas não alcançou `QuotaCounter.tsx`

### Fix Sugerido
```tsx
// QuotaCounter.tsx:96
// ANTES:
{isFreeTier ? "Período de avaliação" : `Renovação: ${formattedResetDate}`}
// DEPOIS:
{isFreeTier ? "Período de avaliação" : `Reset em: ${formattedResetDate}`}
```
