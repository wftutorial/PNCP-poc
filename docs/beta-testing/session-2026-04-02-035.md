## ISSUE-056 — ⚠️ PERSISTE (P2, 8ª sessão consecutiva)

**Persona:** Jordan (Security Mindset)
**Severidade:** P2
**Área:** Admin
**Status:** ABERTO — sem progresso desde sessão 028

### Comportamento Observado
- Console: `Failed to load resource: 503 @ https://smartlic.tech/api/admin/metrics`
- UI: "Backend metrics indisponível: METRICS_TOKEN configurado no frontend mas rejeitado pelo backend. Verifique se o valor coincide com a variavel METRICS_TOKEN no servico backend (railway variables --service bidiq-backend)."
- Fixes anteriores (.strip(), .trim()) não resolveram.

### Root Cause (diagnóstico pendente)
Os tokens provavelmente são genuinamente diferentes entre os dois serviços Railway. Não é problema de whitespace.

### Fix (1 minuto, CLI)
```bash
# Verificar os valores EXATOS em cada serviço:
railway variables --service bidiq-backend | grep METRICS_TOKEN
railway variables --service bidiq-frontend | grep METRICS_TOKEN
# Copiar o valor do backend para o frontend (ou vice-versa):
railway variables set METRICS_TOKEN=<valor-do-backend> --service bidiq-frontend
```

## ISSUE-066 — ABERTO (P3)

**Persona:** Alex "The Closer"
**Severidade:** P3
**Área:** Login / Auth
**Data:** 2026-04-02
**Status:** ABERTO

### Passos para Reproduzir
1. Estar autenticado (qualquer conta)
2. Navegar para `https://smartlic.tech/login`
3. Observar os toasts na tela

### Comportamento Observado
3 toasts simultâneos com mensagem "Você já está autenticado!" aparecem ao mesmo tempo. Página mostra corretamente o estado "Você já está logado" com o email e botão "Ir para o painel".

### Comportamento Esperado
1 toast apenas, ou nenhum toast (a mensagem na tela é suficiente).

### Notas Técnicas
Provavelmente um useEffect ou middleware que dispara o toast em múltiplos lugares (ex: Supabase onAuthStateChange + middleware de rota + componente de login). Cada render/verificação chama `toast()` independentemente.

### Screenshot
`docs/beta-testing/screenshots/session-035/` (documentado via Playwright snapshot — 3 `listitem` com "Você já está autenticado!" no `region "Notifications"`)

### Fix Sugerido
Verificar `app/login/page.tsx` e `middleware.ts` — garantir que o toast "já autenticado" seja disparado em apenas um lugar, com deduplicação via `toast.id` fixo (ex: `toast("Você já está autenticado!", { id: "already-auth" })`).
