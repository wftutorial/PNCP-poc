## ISSUE-036: Filtros "Encerradas" e "Em Julgamento" retornam 0 resultados — data lake so contem licitacoes vivas

**Persona:** Priya "The Edge Lord"
**Severidade:** P1
**Area:** Buscar — Filtros de Status
**Data:** 2026-03-29 14:02
**Status:** ABERTO

### Passos para Reproduzir
1. Login em https://smartlic.tech/buscar
2. Selecionar setor "Engenharia, Projetos e Obras", UF: SP
3. Expandir filtros avancados, selecionar "Encerradas"
4. Clicar "Buscar"

### Comportamento Observado
- "Encontramos 373 licitacoes, mas nenhuma passou nos filtros para Engenharia, Projetos e Obras"
- 0 resultados para "Encerradas" e provavelmente 0 para "Em Julgamento"

### Comportamento Esperado
- Filtro "Encerradas" deveria retornar licitacoes ja homologadas/concluidas
- Filtro "Em Julgamento" deveria retornar licitacoes em fase de julgamento
- Se o data lake so tem licitacoes vivas, o sistema deveria: (a) fazer consulta live na PNCP para esses status, OU (b) informar claramente ao usuario que esses filtros so funcionam com consulta live, OU (c) desabilitar os filtros que nao tem dados

### Notas Tecnicas
- Confirmado pelo usuario: "data lake e populado apenas com licitacoes vivas (correto). licitacoes expiradas tem de ser consulta live"
- O filtro de status FUNCIONA tecnicamente (ISSUE-033 resolvido) — o problema e que nao ha dados de licitacoes encerradas no data lake
- A PNCP API suporta parametro de status, mas o pipeline atual nao diferencia a consulta por status

### Fix Sugerido
Opcoes (mutuamente exclusivas):
1. **Consulta live**: Quando status != "abertas", fazer consulta direta na PNCP com filtro de status server-side (mais complexo, mais dados)
2. **UX honesta**: Adicionar tooltip/aviso nos radios "Em Julgamento" e "Encerradas" explicando que requerem consulta live (mais rapido)
3. **Desabilitar**: Marcar "Em Julgamento" e "Encerradas" como "Em breve" se consulta live nao esta implementada

## ISSUE-037: Cache restore mostra resultados de busca anterior com setor errado no dropdown

**Persona:** Alex "The Closer"
**Severidade:** P2
**Area:** Buscar — Cache/Estado
**Data:** 2026-03-29 14:08
**Status:** ABERTO

### Passos para Reproduzir
1. Login em https://smartlic.tech/buscar
2. Selecionar "Engenharia, Projetos e Obras", buscar
3. Mudar para "Vestuario e Uniformes" no dropdown (sem buscar)
4. Navegar para outra pagina (ex: /pipeline)
5. Voltar para /buscar

### Comportamento Observado
- Toast: "Resultados da analise anterior restaurados"
- Dropdown setor mostra "Vestuario e Uniformes"
- Resultados mostram licitacoes de ENGENHARIA
- Filtros ativos mostram "Vestuario e Uniformes"
- Resumo IA diz "setor de Engenharia, Projetos e Obras"
- Contradicao total entre form state e resultados exibidos

### Comportamento Esperado
Ao restaurar cache, o dropdown e filtros ativos devem refletir o setor da busca cacheada, nao o ultimo estado do formulario. Ou nao restaurar se o formulario mudou.

### Fix Sugerido
Salvar o setor junto com os resultados no cache. Ao restaurar, comparar setor cacheado com setor atual do formulario — se diferente, nao restaurar (forcar nova busca).
