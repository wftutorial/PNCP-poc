## Observacoes Tecnicas Criticas

### ISSUE-025 — Root Cause Analysis

O fix `9111dde3` fez duas mudancas:
1. **RC1**: Eliminou `KEYWORDS_UNIFORMES` como default quando `keywords=None` (mudou para `set()`)
2. **RC2**: Baixou `TERM_DENSITY_LOW_THRESHOLD` de 1% para 0.3%

**Por que nao resolveu:**
- A busca Engenharia+MG foi FRESCA (POST retornou 0 inicialmente, buscou ao vivo do PNCP)
- 413 licitacoes de MG foram encontradas, ZERO passaram no filtro de Engenharia
- O fix RC1 elimina keywords vazias como default — mas no modo SETOR, as keywords existem (sao do YAML)
- Hipotese: As keywords do setor Engenharia no `sectors_data.yaml` simplesmente nao matcham nenhuma das 413 licitacoes. Possivel que as keywords sejam muito especificas/restritas ou que haja um bug na funcao de matching (case sensitivity? word boundary? regex?)
- **ACAO NECESSARIA**: Dump debug das 413 licitacoes vs keywords do setor Engenharia para ver por que 0% match

### ISSUE-021 — Pipeline Persistencia

- POST `/api/pipeline` agora retorna 201 (antes 409) — fix parcial
- GET `/api/pipeline?limit=200` retorna 200 mas com 0 itens
- Hipotese: INSERT no Supabase funciona (201) mas a query de leitura nao encontra o item — possivel RLS policy restritiva ou campo de filtro incorreto no SELECT
