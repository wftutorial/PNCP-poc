## Acceptance Criteria

### Padrão de Qualidade — Todos os Artigos

- [x] **AC1:** Cada artigo tem entre 2.000 e 3.500 palavras de conteúdo substantivo
- [x] **AC2:** Cada artigo segue hierarquia H1 → H2 → H3 sem pulos
- [x] **AC3:** Cada artigo contém no mínimo 3 dados/estatísticas verificáveis com fonte
- [x] **AC4:** Cada artigo contém no mínimo 1 framework/template prático (box destacado)
- [x] **AC5:** Cada artigo termina com seção FAQ (3-5 perguntas) com FAQPage schema JSON-LD
- [x] **AC6:** Tom editorial: institucional, sóbrio, consultivo. Fala DE CONSULTOR PARA CONSULTOR
- [x] **AC7:** Nenhuma afirmação sem fundamento. Dados sem fonte devem ser "estimativa baseada em [metodologia]"

- [x] **AC8:** Cada artigo tem `metadata: Metadata` com title (≤60 chars), description (150-160 chars), canonical URL, OG tags
- [x] **AC9:** Cada artigo tem BlogPosting JSON-LD: headline, description, author, publisher, datePublished, dateModified, wordCount, articleSection, inLanguage
- [x] **AC10:** Cada artigo tem BreadcrumbList JSON-LD: Início > Blog > Consultorias > [Título]
- [x] **AC11:** Cada artigo tem FAQPage JSON-LD na seção FAQ
- [x] **AC12:** URL slug SEO-friendly: lowercase, hifenizado, sem stop words, ≤60 chars
- [x] **AC13:** Keyword primária aparece no H1, first paragraph, e meta description
- [x] **AC14:** Alt text descritivo em todas as imagens/diagramas (se houver)
