### ISSUE-SEO-004: `/glossário` (com acento) retorna 404 — slug real é `/glossario`

**Persona:** Jordan
**Severidade:** P2 — Important
**Área:** SEO / 404 / Redirect
**Data:** 2026-04-08
**Status:** ABERTO

### Comportamento Observado

O GSC reporta `https://smartlic.tech/glossário` (com acento no "a") como **404**. Verificação direta:

```
curl https://smartlic.tech/gloss%C3%A1rio → 404
curl https://smartlic.tech/glossario    → 200
```

O sitemap usa corretamente `/glossario` (sem acento). A URL com acento foi encontrada pelo Googlebot via algum link externo ou interno que usa o slug acentuado.

### Impacto

- 1 URL morta no relatório de 404 do GSC
- Se alguém compartilhou um link com `/glossário`, todos esses visitantes chegam em 404
- Pequeno mas fácil de corrigir

### Fix Sugerido

Adicionar redirect 301 de `/glossário` → `/glossario` no Next.js (`next.config.js`):

```js
redirects: async () => [
  {
    source: '/gloss%C3%A1rio',
    destination: '/glossario',
    permanent: true,
  },
  {
    source: '/glossário',  // Next.js também aceita unicode direto
    destination: '/glossario',
    permanent: true,
  }
]
```

Também investigar qual página/link usa `/glossário` com acento e corrigir na origem.

## Análise de Saúde do SEO — Diagnóstico Completo

### O que está funcionando ✅

1. **ISSUE-SEO-001 RESOLVIDA:** Google-Extended agora tem `Allow: /` explícito na seção SmartLic (override do Cloudflare Managed block). RFC 9309 §2.2.2 garante que Allow vence Disallow na mesma especificidade.

2. **ISSUE-SEO-002 RESOLVIDA:** Sitemap limpo — 0 URLs com `?modalidade=`. Redução de 3.923 → 2.303 URLs legítimas. Crawl budget preservado.

3. **Ondas 1-3 sendo detectadas:** 569 páginas em fila de indexação — sinal positivo. Google detectou todas via sitemap e está enfileirando para crawl.

4. **Tráfego orgânico emergindo:** Queries como "licitação de projetos de engenharia 2026" e "licitação desenvolvimento de software 2026" gerando impressões — Onda 2 funcionando.

5. **Sitemap processado:** Status "Processado" no GSC. Última leitura: 7 abr. Próxima re-leitura esperada: ~14 abr (após Google detectar mudanças).

6. **robots.txt correto:** `/api/og` bloqueado corretamente, Googlebot `*` com Allow: / e Disallow nas rotas autenticadas.

### O que preocupa ⚠️

1. **569 "Detectada, mas não indexada":** Normal para novo site com expansão massiva de sitemap. Mas o Google pode manter nesta categoria por semanas/meses se o conteúdo for percebido como thin. Prioridade: aumentar qualidade e unicidade do conteúdo pSEO.

2. **9 "Rastreada, mas não indexada":** 4 são ?modalidade= artifacts (resolvem sozinhos após re-crawl). 5 são páginas pSEO reais com conteúdo potencialmente thin. Google rastreou e decidiu não indexar — sinal de qualidade insuficiente.

3. **CTR=0% em páginas com alta impressão:** `/blog/pncp-guia-completo-empresas` (153 imp), `/blog/como-participar-primeira-licitacao-2026` (68 imp). Conteúdo aparece mas não converte em cliques.

### O que é ruído (não precisa de ação)

- **3 "404" no GSC:** `/glossário` é real (P2 fácil). `www.smartlic.tech/` já tem 301 redirect — dados stale do GSC de antes do fix.
- **1 "Bloqueada por robots.txt":** `/api/og` — intencional e correto.
- **GSC mostra 2.758 URLs no sitemap:** Leitura pré-fix (7 abr). Irá atualizar na próxima leitura para ~2.303.
