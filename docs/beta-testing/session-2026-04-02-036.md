## ISSUE-067 — ABERTO (P3)

**Persona:** Alex "The Closer" + Sam "The Thumb"
**Severidade:** P3
**Área:** Onboarding
**Data:** 2026-04-02
**Status:** ABERTO

### Passos para Reproduzir
1. Navegar para `https://smartlic.tech/onboarding`
2. Observar o rodapé do card de Step 1 ("Qual é o seu negócio?")

### Comportamento Observado
Dois botões "Pular por agora" renderizados simultaneamente:
- Um à esquerda do rodapé (ref=e33)
- Um no centro do rodapé, ao lado de "Continuar" (ref=e35)

Em mobile 375px o problema é mais visível — os dois botões ficam juntos na mesma linha, antes do "Continuar".

### Comportamento Esperado
Apenas 1 "Pular por agora" — ou à esquerda (sozinho), ou no centro antes de "Continuar". Nunca os dois ao mesmo tempo.

### Notas Técnicas
Snapshot confirmou estrutura:
```
button "Pular por agora" [ref=e33]  ← esquerda
generic:
  button "Pular por agora" [ref=e35]  ← centro
  button "Continuar" [ref=e36]
```
O step 2 só tem 1 "Pular por agora" (correto). O problema é específico do step 1.

### Fix Sugerido
Remover um dos dois botões em `app/onboarding/` — provavelmente o `e33` (esquerda) é um remanescente de layout anterior que não foi removido.

## ISSUE-068 — ABERTO (P3)

**Persona:** Jordan "The Paranoid"
**Severidade:** P3
**Área:** Signup / Auth
**Data:** 2026-04-02
**Status:** ABERTO

### Passos para Reproduzir
1. Estar autenticado
2. Navegar para `https://smartlic.tech/signup`

### Comportamento Observado
A página `/signup` renderiza o formulário de criação de conta normalmente, sem redirecionar o usuário autenticado.

### Comportamento Esperado
Consistência com `/login`: quando usuário autenticado acessa `/login`, é redirecionado para `/buscar` com mensagem "Você já está logado". O mesmo deveria ocorrer em `/signup`.

### Notas Técnicas
O usuário autenticado poderia tecnicamente tentar criar uma 2ª conta pelo formulário — o Supabase retornaria erro se o email existir, mas outros emails poderiam criar contas extras. Baixo risco mas inconsistente.

### Fix Sugerido
Adicionar guard em `app/signup/page.tsx` análogo ao existente em `app/login/page.tsx`:
```typescript
if (session) redirect('/buscar');
```
