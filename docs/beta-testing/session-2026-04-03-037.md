## ISSUE-070 — ABERTO (P2)

**Persona:** Jordan "The Paranoid"
**Severidade:** P2
**Área:** /conta/plano — Billing Info
**Data:** 2026-04-03
**Status:** ABERTO

### Passos para Reproduzir
1. Fazer login como usuário com assinatura ativa
2. Navegar para `https://smartlic.tech/conta/plano`
3. Observar o campo "Próxima renovação"

### Comportamento Observado
"Próxima renovação" exibe `quota_reset_date` = **1º do próximo mês** (reset da cota mensal de análises).
- Para conta admin: exibe `datetime.now()` → data de hoje (ex: "03 de abril")
- Para usuário regular com plano Mensal: exibe "01 de maio" (1º do próximo mês)
- Mas a renovação real do Stripe pode ser em qualquer dia do mês (ex: usuário assinou no dia 15 → renova dia 15)

### Comportamento Esperado
"Próxima renovação" deveria exibir a data real de cobrança do Stripe (`current_period_end`), sincronizada via webhooks na tabela `plan_billing_periods` ou `profiles`.

### Notas Técnicas
- `authorization.py:165`: admins recebem `quota_reset_date=datetime.now(timezone.utc)` → mostra hoje
- `quota.py:469-475`: usuários regulares recebem `quota_reset_date = 1º do próximo mês`
- `frontend/app/conta/plano/page.tsx:82`: label "Próxima renovação" usa `planInfo.quota_reset_date`
- A data real de renovação Stripe está em `plan_billing_periods.period_end` ou `profiles.subscription_end_date`

### Fix Sugerido
**Opção A (rápido):** Renomear label de "Próxima renovação" para "Próximo reset de cota" — semanticamente correto, sem mudança backend.

**Opção B (correto):** Expor `current_period_end` via API (`GET /me`) e usar como "Próxima renovação". Manter `quota_reset_date` separado para exibir "Reset de cota: 01/mai".
